  一、Learnable Label Embeddings（可学习标签嵌入）

  核心设计思想

  MSQNet 的标签嵌入不是随机初始化的，而是用 CLIP 文本编码器提取每个动作类别名称的语义向量来初始化，再让其在训练中继续更新。

  初始化过程（_get_text_features）

  文件： query2labelclipinit.py:64-70 / timesformerclipinitvideoguide.py:71-77

  def _get_text_features(self, cl_names):
      text_model = CLIPTextModel.from_pretrained("openai/clip-vit-base-patch16")
      tokenizer  = CLIPTokenizer.from_pretrained("openai/clip-vit-base-patch16")
      act_prompt = self._get_prompt(cl_names)          # ["running", "jumping", ...]
      texts      = tokenizer(act_prompt, padding=True, return_tensors="pt")
      text_class = text_model(**texts).pooler_output.detach()  # [K, 512]
      return text_class

  - 输入：class_list，即所有动作类别的文字名称
  - 过程：逐类别 tokenize → CLIP Text Encoder → 取 pooler_output（CLS token 的池化表示）
  - 输出：形状为 [K, D]（K = 类别数，D = CLIP 隐层维度，如 512）
  - 关键：.detach() 表示 CLIP 文本编码器此时不参与梯度反传，仅用于初始化

  二、Multi-modal Fusion（多模态融合）

  核心问题：Query 如何融合两种模态？

  MSQNet 的多模态融合发生在 Query 构造阶段，在 Transformer Decoder 之前，将文本语义（标签嵌入）与视觉特征（CLIP 图像编码器）融合，生成"多模态查询"。

  融合的具体实现

  文件： timesformerclipinitvideoguide.py:26-35 / videomaeclipinitvideoguide.py:28-38

  def forward(self, images):
      b, t, c, h, w = images.size()

      # ── 路径1：视频时序特征（视觉骨干）──
      x = self.backbone(images)[0]                          # VideoMAE/TimeSformer → [b, N, hidden]
      x = self.linear1(adaptive_avg_pool1d(x, t))           # 压缩到 t 帧 → [b, t, embed_dim]
      x = self.pos_encod(x)                                 # 加位置编码
    
      # ── 路径2：逐帧 CLIP 视觉特征（图像编码器）──
      video_features = self.image_model(                    # CLIPVisionModel
          images.reshape(b*t, c, h, w)
      )[1]                                                  # [B*T, embed_dim]  取 pooler_output
      .reshape(b, t, -1)
      .mean(dim=1, keepdim=True)                            # 时间维平均 → [b, 1, embed_dim]
    
      # ── 融合：拼接文本嵌入 + CLIP 视觉特征 → linear2 投影 ──
      query_embed = self.linear2(
          torch.concat((
              self.query_embed.unsqueeze(0).repeat(b, 1, 1),  # 文本语义：[b, K, embed_dim]
              video_features.repeat(1, self.num_classes, 1)    # 视觉语义：[b, K, embed_dim]
          ), dim=2)                                            # 拼接 → [b, K, embed_dim*2]
      )                                                        # 经 linear2 → [b, K, embed_dim]
    
      # ── Transformer Decoder：Query 与视频特征做注意力 ──
      hs = self.transformer(x, query_embed)                  # [b, K, embed_dim]
      out = self.group_linear(hs)                            # [b, K] 最终分类 logits
      return out

  GroupWiseLinear 的作用

  文件： videomaeclipinitvideoguide.py:178-206

  def forward(self, x):
      # x: [B, K, D]  — 每个类别对应一个 D 维向量
      x = (self.W * x).sum(-1)   # 每类独立做点乘 → [B, K]
      if self.bias:
          x = x + self.b
      return x

  这是类别独立的线性分类头，每个类别 k 有自己专属的权重向量 W[0][k]，而非所有类别共享同一 Linear 层，更适合多标签分类场景。
