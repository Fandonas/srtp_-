  🎯 两种索引方式的对比

  ❌ 错误方式：使用全局索引

  gt_index = 23  # "catch"的全局ID
  logits = -class_cum_dists  # [5]，只有5个类别的距离

  # 问题：logits只有5个元素，但gt_index=23超出范围！
  loss = F.cross_entropy(logits.unsqueeze(0), torch.tensor([23]))
  # 报错：IndexError: Target 23 is out of bounds.

  为什么错误：
  - logits 的维度是 [num_episode_classes]，在5-way任务中只有5个元素
  - 如果 gt_index=23，PyTorch会尝试访问 logits[23]，但 logits 只有索引0-4
  - 这会导致索引越界错误

---
  ✅ 正确方式：使用相对索引

  # 当前episode的类别列表
  episode_classes = ["brush hair", "catch", "clap", "climb", "dive"]
  #                   索引0          索引1    索引2    索引3     索引4

  # 真实类别在episode中的相对位置
  gt_name = "catch"
  gt_index = episode_classes.index(gt_name)  # = 1（相对索引）

  # logits的构建顺序与episode_classes一致
  logits = torch.tensor([
      -dist("brush hair"),  # logits[0]
      -dist("catch"),       # logits[1] ← 真实类别
      -dist("clap"),        # logits[2]
      -dist("climb"),       # logits[3]
      -dist("dive")         # logits[4]
  ])  # shape: [5]

  # 交叉熵计算
  loss = F.cross_entropy(logits.unsqueeze(0), torch.tensor([1]))
  # 等价于：-log(softmax(logits)[1])
  # 即：最大化"catch"类别的概率

  为什么正确：
  - logits[i] 对应 episode_classes[i] 的预测分数
  - gt_index=1 指向 logits[1]，即"catch"类别的分数
  - 索引范围匹配：gt_index ∈ [0, 4]，logits.shape[0] = 5

---
  🧠 Few-shot学习的本质

  在Few-shot学习中，每个episode是一个独立的子任务：

  # Episode 1: 5个类别的分类任务
  episode_classes = ["A", "B", "C", "D", "E"]  # 从100个类别中采样
  # 模型需要在这5个类别中做分类

  # Episode 2: 另外5个类别的分类任务
  episode_classes = ["F", "G", "H", "I", "J"]  # 不同的5个类别
  # 模型需要在这5个类别中做分类

  关键点：
  - 每个episode的分类空间是动态变化的
  - 模型输出的logits维度 = 当前episode的类别数
  - 标签索引必须是相对于当前episode的

  📐 数学原理验证

  Cross Entropy的定义

  loss = -log(softmax(logits)[gt_index])
       = -log(exp(logits[gt_index]) / Σ_i exp(logits[i]))

  要求：
  - gt_index 必须是 logits 的有效索引
  - 0 ≤ gt_index < len(logits)

  当前实现

  # 满足条件：gt_index是logits的有效索引 ✅

---
  🎓 设计原理总结

  使用相对索引的原因：

  1. 维度匹配：logits 的大小等于episode类别数，不是全局类别数
  2. 动态任务：每个episode的类别集合不同，无法使用固定的全局索引
  3. 语义对应：logits[i] 的语义是"当前视频属于 episode_classes[i] 的概率"
  4. Few-shot本质：模型需要在当前episode的类别子集中做分类，而不是全局类别空间
