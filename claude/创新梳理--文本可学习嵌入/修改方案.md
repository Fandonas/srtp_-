---
  一、维度等价性确认
                                                                                                                                                                                           
  现有 context_support:
    text_features_train[support_real_class.long()].unsqueeze(1)
    → [support_bs, 1, 512]  ← 冻结，无梯度

  拟用 query_embed:
    self.query_embed(support_real_class).unsqueeze(1)  (或直接索引 weight)
    → [support_bs, 1, 512]  ← 可学习，有梯度

  两者形状完全相同，可以直接替换，后续 torch.cat 和 context2 一行代码都不用动。

---
  二、参数差异的本质

                       梯度  |  训练中更新  |  初始值
  context_support(冻结)   ✗  |      ✗      |  CLIP文本特征（逐 episode 动态取）
  query_embed(可学习)     ✓  |      ✓      |  CLIP文本特征（一次性 nn.Parameter）

  context_support 相当于每次从字典里查固定值；query_embed 相当于这个字典本身在训练中被反向传播持续修正——它会朝着"更适合做 support
  增强的语义向量"方向漂移，而不是停留在纯语言模型的文本空间。

---
  三、构造方式（参照 MSQNet 的完整写法）

  # __init__ 中（参考 MSQNet query2labelclipinit.py:19-20）

  # Step1: 用 CLIP 文本特征初始化
  class_embed_train = self.text_features_train  # [num_train_classes, 512]，已冻结的CLIP输出

  # Step2: 包装成可学习 Embedding
  self.query_embed_train = nn.Embedding(num_train_classes, self.mid_dim)
  self.query_embed_train.weight = nn.Parameter(class_embed_train.clone())
  #  ↑ .clone() 断开与 text_features_train 的引用关系，后者保持冻结

  # forward 中替换原来的 context_support 一行：
  # 原：context_support = self.text_features_train[support_real_class.long()].unsqueeze(1)
  # 新：
  context_support = self.query_embed_train.weight[support_real_class.long()].unsqueeze(1)
  # → [support_bs, 1, 512]，从这里开始有梯度

---
  四、训练 vs 测试 的关键分裂问题

  这是此方案最需要注意的地方：

  训练阶段:
    support_real_class ∈ [0, num_train_classes-1]
    → 可以索引 query_embed_train.weight，有梯度 ✓

  测试阶段（novel classes）:
    support_real_class ∈ [0, num_test_classes-1]
    → 这些类从未见过，query_embed_train 里根本没有对应行 ✗

  所以测试时必须回退到冻结的 CLIP 特征：

  if self.training:
      context_support = self.query_embed_train.weight[support_real_class.long()].unsqueeze(1)
  else:
      context_support = self.text_features_test[support_real_class.long()].unsqueeze(1)
      # 测试类是 unseen，无法预训练 embedding，仍用冻结 CLIP

  这与 few-shot 学习的标准范式一致：训练类上学习"如何用文本辅助视觉匹配"，测试类依赖 CLIP 泛化能力。

---
  五、mid_layer 的角色重新审视

  目前 mid_layer = nn.Sequential()（恒等映射，few_shot.py:2744），对 context_support 不做任何变换。

  引入 query_embed 后，由于参数本身已可学习，mid_layer 可以保持不变。但若想让 query_embed 经过一层非线性投影再 cat（类似 MSQNet 的 MLP Fusion），则可以在 mid_layer
  里加一个线性层——这属于进一步的改动，不是必须的。
